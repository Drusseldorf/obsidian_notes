# 4.28-29 ПРАВА ДОСТУПА

```
ФАЙЛЫ

	000 - ничего нельзя (0)
	001 - можно только исполнять (1)
	010 - только изменять (2)
	100 - только читать (4)
	111 - все можно (7)

	chmod 7(владелец)7(группа)7(все остальные)

	три вида прав на любой файл: r,w,x - прочитать, записать, исполнить.
	всегда имеется пользователь который владеет файлом, группа которая владеет файлом, и все остальные

	ls -l выведет список и слева будут написаны права доступа(три группы символов) :
	первый символ - тип файла
	след. 3 - права для владельца
	след. 3 - права для группы, владеющей файлом
	след. 3 - права для остальных

	rw-  можно читать, изменять, нельзя исполнять (всегда порядок такой rwx)

	chmod o-r filename (o - others, -r - убрать права на r для остальных)
	chmod u+x filename (u - user, +x - добавили право на исполнение для user)
	chmod g+rwx filename (g - group, +rwx добавили все права для владельца группу)

	sudo chgrp groupname filename (изменяет группу-владельца) или chown

	Изменить владельца юзера и или группу:

	sudo chown user filename (or dir name) - меняет оунера
	sudo chown group filename (or dir name) - меняет группу
	sudo chown user:group file_or_dir - меняет сразу и владельца и группу
	sudo chown :group somefile.txt — смена группы, владеющей файлом, на group
	sudo chown user: file_or_dir - группа поменяется на главную группа пользователя user

ДИТРЕКТОРИИ

	r - можно делать ls для этой директории
	w - можно создавать файлы, изменять, переименовывать, удалять файлы
	x - можно входить в директорию, смотреть атрибуты файлов (ls -la), получать доступ к содержимому файлов директории (даже если есть доступ к файлу, все равно должен быть x для директории где файл лежит)

	аналогично через chmod o-r foldername - меняем права

	если нет права на x, но выполнить ls -ls то получим ???? вместо атрибутов
```
